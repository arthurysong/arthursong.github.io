Console-Poker (WIP)

Console-Poker is a Poker website that I built because my friends and I enjoy playing poker when we hangout, and what better time to create a virtual game then now during COVID! I knew there are many poker games out there so I wanted to create my own take by creating a UI where players play poker using a terminal, similar to how coders execute commands in a terminal--I think working with a terminal is very appealing, and I think with the popularity of coding in media like Mr. Robot and Silicon Valley even non-coders find terminals very fascinating as well. Note: Later on in my project implementation, after receiving feedback from people, I decided to remove the terminal UI because it made the game less accessible. Users would have to read through a bunch of console messages to determine the state of the game. The terminal also seemed inconsistent with the (Nes) style of the UI.

What are some things I was thinking about when I created this project?
1. I knew I wanted a bank feature, where user's deposit funds into their account because the best part about playing Poker is playing with real money. This depositing and withdrawing feature I implemented using Stripe!

2. Another thing I thought about was I wanted to have a specific look for my project. Since, the terminal was going to be central to playing the game, I wanted the style or theme to feel retro. I found this style sheet called Nes.css with 8 bit styling, which I thought would be a great look to the app.

3. Finally, I wanted socializing or chatting to be central to the application. I felt like many online poker sites out there minimalize the social aspect of playing poker. When I'm playing poker with my friends, part of the fun is being able to talk and banter with each other. Therefore, for my application, I wanted the chatting aspect of the game to be of equal importance to the Poker aspect of the game. To achieve this, I set out for my chatting UI to be bigger, and also more interesting to look at (messages are decorated within a themed chatbox and encapsulated within bubbles).

Challenges
The biggest challenge I had going into this application was implementing Real Time communication between Client and Database (WebSockets). I had never implemented a WebSockets connection and it was interesting learning about and implementing ActionCable to manage the connection. Many features of my application required WebSockets connection: chatting, playing Poker, and keeping an updated list of lobbies. The other big challenge I encountered was designing the Poker logic. I created a blog post where I talk about the key things to creating the models and persisting the game of Poker to a database. All in all, it was very fun learning how to implement these challenges.

Now without further ado, here is a walkthrough of the application
1. Sign in / Register
 Sign in to an existing account, or register a new account! Users can also sign in using a registered Google account.


2. Deposit Money
 Navigate to the 'Bank' and buy chips with USD using a Credit/Debit card. 1 USD = 10000 Chips.


3. Join a Room and Play
 Go back to the Rooms page, join or create a new room with your friends, and start playing Poker. You can have fun chatting as well!


4. Withdraw your Winnings!
 Once you and your friends are done playing, navigate back to the Bank, click withdraw. First time users need to create/connect a Stripe Connect account (using either a bank account or credit/debit card). Once connected, exchange and withdraw your available chips into the connected account.


And that's it!

Technologies used in this application are:
  •  RESTful Rails API with technical models/associations
  • JWT authentication, OmniAuth authentication with Google
  • Rails ActionCable channels for WebSocket Connections
  • React Frontend with Redux for state management
  • ActionCable connection/subscriptions managed using Redux middlewares
  • Graphic UI to display Poker game
  • Stripe Implementation for User deposits/withdrawals
  • Retro CSS styling using Nec.CSS stylesheet

iManage

iManage is a CMS React/Rails application I designed to help my apartment manager manage the residents and units in her apartment complex. I was inspired by the application I use to pay rent online, and wanted to create a mock version of that application. Basically, I wanted users to be able to log in, view their balance, and pay their balances from the comfort of their own home.

This project was mostly an application of the concepts/skills I learned about React and Redux state management!

Walkthrough of iManage
There are two different types of users: manager and tenants. Manager can log in to their account and see all the units and users in the application. The manager can assign and remove users from a unit, creating or destroying "residencies." Once a user has been assigned to a unit, the application will calculate the balance for the user depending on how much the unit costs, how many months have passed, and also the users' past payments.

Finally, a resident that has been assigned to a unit can log in to their account, check out past payments, and also submit a new payment towards their balance.

Manager Web Flow
1. Manager Sign In
Manager signs into her account.
2. View Units
View the units in her apartment complex. See what tenants are living in each unit. Check to see if they have paid their balance. If manager needs to contact a unit's tenant, find their numbers here.
3. Create a residency/Destroy residency
Click on an empty unit, and add an existing user to that unit. The unit will start to accrue balance. When a user leaves the apartment complex, simply remove them from them residency.
4. View Registered Users
See the users that have registered on the site, and check out their information.
Tenant Web Flow
1. Tenant Register & Sign In
New users register an account and sign in to their account.
2. View balance and Make Payment
Once a user has been assigned to a unit, users can submit a payment towards their balance. Users can also view their past payments.
Finish!

Technologies used in this application are:
  • RESTful Rails API with CRUD/Validations/Associations
  • React Front end with Redux & Thunk
  • Authentication using Rails Sessions/Redux/Bcrypt
  • Client-Sided Routing using React-Router-Dom
  • Component Lifecycle Methods used for Authorizing User Routes & Loading Resources
  • Basic styling using React-Bootstrap and CSS
